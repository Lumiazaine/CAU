<panel type="info" title="4. Flujo de Tareas">


En algunos casos, durante la gestión de tickets en la categoría **Adriano**, se detecta que el usuario final requiere **soporte formativo adicional**. Para ello, se ha configurado un flujo mediante **plantilla de orden de trabajo** (WO) para dar seguimiento y gestión a este tipo de necesidad.

==== Plantilla de WO configurada ====
Desde **"Plantilla de orden de trabajo"**, buscamos por:

  * **Nombre de plantilla**: Petición de Soporte formativo Adriano
  * **Resumen**: Petición de Soporte Formativo.
  * **Grupo de asignación**: Servicios/Capacitación (se asigna automáticamente).
  * **Tipo de tarea**: Orden de Trabajo (WO) generada desde ticket principal

{{ :pag_hgis:adri1.png?400 |}}

==== ¿Cuándo se genera esta WO? ====
La plantilla se activa **manualmente** cuando, durante el tratamiento de un ticket principal en la categoría Adriano, cuando se identifica que:
  * El usuario necesita formación específica para utilizar el sistema Adriano correctamente.
  * El técnico no puede resolver la petición directamente por falta de conocimiento del usuario final.

En ese caso:
  * El **ticket principal se puede cerrar** con el **motivo de estado de cierre**: **Imposible ejecución**
  * Se **genera la WO** a través de la plantilla mencionada para ser gestionada por el grupo de capacitación.

==== Objetivo de la tarea ====
Derivar correctamente las necesidades de capacitación del usuario a **Servicios/Capacitación**, asegurando trazabilidad y separación entre:
  * Petición técnica no ejecutable.
  * Necesidad formativa que requiere seguimiento independiente.

==== Consideraciones ====
  * Esta orden de trabajo debe ir **asociada al ticket original** (relación padre-hijo).
  * El grupo *Servicios/Capacitación* realizará el seguimiento formativo correspondiente.
  * Este proceso mejora la visibilidad de las necesidades formativas detectadas desde ITSM.
