====== Facturación ======

Datos aclaratorios

Cuando iniciemos la facturación el primer paso será crear un lote (en la opción nuevo lote de la aplicación consumible) con el archivo xml que hemos recibido del proveedor. En ese proceso realizaremos dos accines que producen:

  * Cuando damos a la opción de CARGAR creamos una tabla en la base de datos denominada ALBARANES XML

  * Cuando damos a la opción de IMPORTAR creamos una tabla en la base de datos denominada LINEA ALBARANES XML

En esas tablas hay un valor de referencia denominado IDALBARANES, este valor es distinto para la tabla ALBARANES del de la tabla ALBARANES XML, por lo que para cualquier modificación en dichas tablas hay que coger el valor IDALBARANES correspondiente.

El IDPEDIDO es el mismo para las tablas ALBARANES que para la tabla ALBARANES.

Cambiar precio de un producto

Coger dato IDALBARAN e IDPEDIDO y ver las tablas siguientes donde modificaremos su precio:

  * LINEA PEDIDO
  * LINEA ALBARANES
  * LINEA ALBARANES XML

====== Error reasignar pedido ======

Cuando el usuario indique que la aplicación le impide realizar un nuevo pedido por que el anterior no esta recepcionado, la solución directa seria que el propio usuario lo recepcionace con el número de albaran, si este no lo tuviese se lo podemos proporcionar nosotros (se lo pediríamos al proveedor) o incluso recepcionarlo desde la misma aplicación, situándonos en el pedido e introduciendo el numero de albaran en el campo con dicho nombre que estará vació o pulsando un doble clic sobre el número de albaran ( que debería estar en blanco o con un numero aleatorio que crea el sistema) y en la pantalla que nos aparece introducir el número de albaran en el campo de albaran.

Si tras esta operación el numero ya estuviese introducido o el usuario indica que le sigue dando error podemos entrar en la base de datos y con el número de pedido y desde la tabla ¿PEDIDOS¿ modificar el estado de dicho registro a el estado 11 (RECEPCIONADO)

</panel></WRAP>

====== No puede recepcionar un pedido ======

Descripción de la incidencia

No puede recepcionar el pedido con el nº de albaran facilitado pro el Dpto de Suministros, indica nº de albaran erroneo.

Solución

1) Localizamos el numero del pedido (X_PEDIDOS) :

SELECT * FROM SUMINISTROS.PEDIDOS

WHERE X_PEDIDO = 'X_PEDIDO';

//– Cogemos USU_X_USUARIO//

2) Buscamos el pedido:

SELECT DISTINCT ART_C_ARTICULO, ALB_C_ALBARAN, PED_X_PEDIDO

FROM SUMINISTROS.recepciones r, SUMINISTROS.pedidos p

WHERE r.ped_x_pedido = p.x_pedido

AND p.d_estado !='CAN'

AND p.usu_x_usuario = 'USU_X_USUARIO'

AND r.art_c_articulo = 'C_ARTICULO'

AND r.alb_c_albaran NOT IN

(SELECT a.c_albaran FROM SUMINISTROS.albaranes_ans a, SUMINISTROS.pedidos p1

WHERE a.ped_x_pedido = p1.x_pedido

AND p1.usu_x_usuario = 'USU_X_USUARIO'

AND a.art_c_articulo = 'C_ARTICULO'

AND p1.x_pedido = 'X_PEDIDO');

//—–Si aparece es que NO tiene el albarán subido.//

//Se busca el pedido en la tabla albaranes_ans, si no está, //indicar que no está cargado el albarán del excel, por eso sale ese error.

En el caso que no aparezca seguimos

2) Si se ha subido correctamente tendría que salir en esta tabla

SELECT a.c_albaran FROM SUMINISTROS.albaranes_ans a, SUMINISTROS.pedidos p1

WHERE a.ped_x_pedido = p1.x_pedido

AND p1.usu_x_usuario = 'USU_X_USUARIO'

AND a.art_c_articulo = 'C_ARTICULO'

AND p1.x_pedido = 'X_PEDIDO';

3) Comprobamos el albarán

select * from suministros.albaranes_ans where c_albaran = 'C_ALBARAN';

Si esta indicar que el alabarán esta subido correctamente y que el usuario que pruebe de nuevo para descartar un error puntual.
