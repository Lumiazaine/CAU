     <div class="block">

     
         <p>SonarQube es una plataforma de código abierto utilizada para analizar la calidad del código fuente de aplicaciones.</p>
         <p>Su principal objetivo es detectar errores, vulnerabilidades de seguridad y malas prácticas en el código.</p>
         <p>Se utiliza durante el desarrollo de software para mantener estándares de calidad y facilitar el mantenimiento.</p>
         <p>SonarQube es compatible con múltiples lenguajes de programación como Java, Python, JavaScript y C#.</p>
         <p>Además, se integra fácilmente con herramientas de integración continua como Jenkins, GitLab o GitHub Actions.</p>
         <p>Gracias a sus reportes y métricas, ayuda a los equipos a mejorar la fiabilidad y seguridad del software.</p>

  <div class="block">
    <h2>4.Sonar WSO2 Profile</h2>



    <p><strong>Lenguaje:</strong> Synapse XML</p>
    <p><strong>Objetivo:</strong> Validación del proyecto WSO2 Microintegrator mediante reglas de Sonar</p>



    <h3>Reglas activadas mediante Xpath</h3>
<table>
  <tr>
    <th>Regla</th>
    <th>Descripción</th>
  </tr>

  <tr>
    <td>Nombre del proyecto</td>
    <td>El proyecto debe tener un nombre con la estructura: Servicios[NOMBRE_DEL_PROYECTO]Parent</td>
  </tr>

  <tr>
    <td>GroupId del proyecto</td>
    <td>El ID de grupo debe ser "es.juntadeandalucia.nija.interoperabilidad"</td>
  </tr>

  <tr>
    <td>Versión del proyecto</td>
    <td>La versión del proyecto debe seguir la estructura: X.Y.Z</td>
  </tr>
  
  <tr>
    <td>Atributo context de la API</td>
    <td>El atributo context en el nodo api debe cumplir el patrón /vX/${variable}</td>
  </tr>
  
  <tr>
    <td>Nombre de la API</td>
    <td>El atributo name en el nodo api debe cumplir el patrón [INT/EXT]_[PUB/PRV]_[NOMBRE_DE_LA_API]_vX</td>
  </tr>

  <tr>
    <td>Atributo uri-template</td>
    <td>El atributo uri-template en el nodo resource debe cumplir el patron /$ {variable</td>
  </tr>

  <tr>
    <td>InSequence API</td>
    <td>En cada nodo resource/inSequence debe haber unanodo sequence con el atributo key siguiendo el patrón INSEQUENCE_POST/GET/PUT_NOMBRERECURSO_vX </td>
  </tr>

  <tr>
    <td>OutSequence API</td>
    <td>En cada nodo resource/outSequence debe haber unanodo sequence con el atributo key siguiendo el patrón OUTSEQUENCE_POST/GET/PUT_NOMBRERECURSO_vX</td>
  </tr>

  <tr>
    <td>FaultSequence API</td>
    <td>En cada nodo resource/faultSequence debe haber una nodo sequence con el atributo key siguiendo el patrón FAULTSEQUENCE_POST/GET/PUT_NOMBRERECURSO_vX </td>
  </tr>

  <tr>
    <td>Nombre proxy service</td>
    <td>El atributo name debe cumplir el patron [INT/EXT]_[PUB/PRV]_[NOMBRE_DEL_PROXY]_vX </td>
  </tr>

  <tr>
    <td>InSequence proxy-service</td>
    <td>En cada nodo target/inSequence debe haber una nodo sequence con el atributo key siguiendo el patrón INSEQUENCE_SOAP_[NOMBRE_DEL_SERVICIO]_vX </td>
  </tr>

  <tr>
    <td>OutSequence proxy-service</td>
    <td>En cada nodo target/outSequence debe haber una nodo sequence con el atributo key siguiendo el patrón OUTSEQUENCE_SOAP_[NOMBRE_DEL_SERVICIO]_vX </td>
  </tr>

  <tr>
    <td>FaultSequence proxy-service</td>
    <td>En cada nodo target/faultSequence debe haber una nodo sequence con el atributo key siguiendo el patrón FAULTSEQUENCE_SOAP_[NOMBRE_DEL_SERVICIO]_vX</td>
  </tr>

  <tr>
    <td>Existencia nodo publishWSDL</td>
    <td>El nodo publishWSDL debe existir</td>
  </tr>

  <tr>
    <td>Atributo key publishWSDL</td>
    <td>El nodo publishWSDL debe contener relleno el atributo key</td>
  </tr>

  <tr>
    <td>Atributo preservePolicy publishWSDL</td>
    <td>El nodo publishWSDL debe contener relleno el atributo preservePolicy con valor a true</td>
  </tr>

  <tr>
    <td>Atributo transport</td>
    <td>El atributo transport debe contener http y https</td>
  </tr>

  <tr>
    <td> Parámetro disableOperationValidation y useOriginalWSDL</td>
    <td>Debe contener los parámetros'disableOperationValidation' y 'useOriginarwsdl', y ambos con el valor 'true'</td>
  </tr>

  <tr>
    <td>Nombre del Endpoint</td>
    <td>El nombre del archivo ubicado en la ruta src/main/wso2mi/artifacts/endpoints/ debe cumplir el patrón [NOMBRE_DEL_ENDPOINT]_EP_vX.xml</td>
  </tr>

  <tr>
    <td>Atributo uri del endpoint</td>
    <td>El atributo uri del nodo endpoint/address atributo uri debe estar relleno por el patrón ${variablei}</td>
  </tr>

  <tr>
    <td>Existencia nodo timeout Endpoint</td>
    <td>Debe existir el nodo timeout/responseAction, con el valor fault</td>
  </tr>

  <tr>
    <td>Existencia nodo timeout/duration Endpoint </td>
    <td>Debe existir el nodo timeout/duration parametrizado</td>
  </tr>

  <tr>

    <td>Debe definirse la acción ante timeout</td>
  </tr>

  <tr>
    <td>Existencia nodo markForSuspension Endpoint</td>
    <td>Verificar si existe el nodo markForSuspension/errorCodes parametrizado</td>
  </tr>

  <tr>
    <td>Existencia nodo markForSuspension/errorCodes Endpoint</td>
