Para realizar el informe técnico de las SEDES accedemos al Report de Oragrid:

- NMC Oragrid → Reports → Reports → NetWorker Backup Status → Group Summary by Client and Server

- Seleccionamos la fecha del día anterior con hora 6:00 y el del día actual con hora actual para los incrementales y desde las 0:00:00 del Sabado hasta las 0:00:00 del Lunes para los backups completos del fin de semana.

- Realizamos dos Reports, uno con todos los Status (failed y succeeded) para contar todos los save sets que hay por grupo.

Si lo exportamos a CSV, solo tenemos que seleccionar en la hoja de cálculo la columna "Number of Save Sets" para que nos de el sumatorio.

También obtenemos la columna de los save sets copiados correctamente.

Todo este informe se copia en la tercera hoja del excel.

Con estos datos rellenamos el informe del día anterior renombrado el archivo con el día de hoy. Este se encuentra en [[\\iunsas05\sij\Sistemas\00\04-Informes\06|]]. Chequeos\Mantenimiento Diario\Informe Tecnico Backup_fecha.ods

En la primera hoja indicamos los save sets correctos y la sede que han fallado.

Verde: 0 fallos

Amarillo: 1 Dia Fallido

Rojo: mas de un día de fallos

Azul: el backup sigue corriendo

Blanco: Ese día no estaba programado el backup

Para indicar la sede podemos recurrir al servidor de dominio (Usuarios y equipos de Active Directory → Cambiar dominio → Examinar → Se indica el domino de la sede → y se busca por el nombre de la sede. Por ejemplo justiciacaroh2 → Dominio Cadiz, Sede Rota-CARO-Celestino Mutis (CARO)

O a la web [[http://infosistemas.justicia.junta-andalucia.es/consultas-escritos/tecnicos/informe.php?f=statspat&type=ROE#|http://infosistemas.justicia.junta-andalucia.es/consultas-escritos/tecnicos/informe.php?f=statspat&type=ROE#]]

En la segunda hoja del excel se guardan los save sets fallidos, que lo obtendremos del reporte "Save Set Details by Client" con la misma fecha del anterior y con el "Status Selected" = "failed"

Para realizar el informe técnico de SSCC accedemos al Report de Iusslnsrsrv01:

- NMC Oragrid → Reports → Reports → Policy Statistics → Workflow Details : Con esto obtenemos el número de save sets fallidos y correctos según el grupo para rellenar la primera hoja del excel

- NMC Oragrid → Reports → Reports → Policy Statistics → Save Sets Details : Con esto obtenemos los save sets fallidos para la tercera hoja del excel

Por otro lado, se verifican los correos que llegan a notificaciones.gtbd o notificaciones.gtab con asunto "Monitorizacion de RMAN ARCH" y "Monitorizacion de EXPORT", para verificar si los exports y Rmans se han creado correctamente y la copia de la bbdd es consistente.
