2. Generar Scripts

Ejemplo: Importar tablas de PRINST de ALEJ a BDADRIANOAUX

Mientras se recupera el DMP, vamos preparando los scripts para crear la infraestructura.

- Entramos en el TOAD y nos conectamos a la BBDD origen = JUSTICIAALEJH1

Tablespaces

- Buscamos los Tablespaces en el Browser →; Tablespaces. Filtramos con PRINST%, seleccionamos todos los tablespaces y
clic derecho →; Create Script →; Screen →; OK.

- Copiamos los scripts (SIN EL DROP STATEMENT) en un bloc de notas propio para modificarlos.

Esquema

- Buscamos el esquema en el Browser →; Users. Filtramos con PRINST, clic derecho en el esquema →; Create Script →; Clipboard →; OK.

- Pegamos el script (SIN EL DROP STATEMENT) al final del bloc de notas propio.

- El nombre de los nuevos tablespaces en BDADRIANOAUX deben llamarse diferente al original y con el siguiente formato: RES_PRINST_ALEJ_XXXX, y también tendremos que renombrar los objetos originales por los que se van a crear en BDADRIANOAUX. 2.2 Modificación de script de Esquema

- El nombre del nuevo esquema en BDADRIANOAUX debe ser diferente al original y con el siguiente formato: RES_PRINST_ALEJ, y también tendremos que renombrar los objetos originales por los que se van a crear en BDADRIANOAUX. Por ello, tendremos que modificar los scripts en el bloc de notas de la siguiente manera:

CREATE USER RES_PRINST_ALEJ
IDENTIFIED BY “RES_PRINST_ALEJ”
DEFAULT TABLESPACE RES_PRINST_ALEJ_DATA
TEMPORARY TABLESPACE TEMP_ADRIANO
PROFILE DEFAULT
ACCOUNT UNLOCK;
– 3 Roles for RES_PRINST_ALEJ
GRANT RESOURCE TO RES_PRINST_ALEJ;
GRANT CONNECT TO RES_PRINST_ALEJ;
GRANT PLUSTRACE TO RES_PRINST_ALEJ;
ALTER USER RES_PRINST_ALEJ DEFAULT ROLE RESOURCE, CONNECT;
– 5 System Privileges for RES_PRINST_ALEJ
GRANT CREATE DATABASE LINK TO RES_PRINST_ALEJ;
GRANT CREATE TABLE TO RES_PRINST_ALEJ;
GRANT UNLIMITED TABLESPACE TO RES_PRINST_ALEJ;
GRANT CREATE MATERIALIZED VIEW TO RES_PRINST_ALEJ;
GRANT CREATE VIEW TO RES_PRINST_ALEJ;

3. Import en BDADRIANOAUX

3.1 Import desde NFS_BACKUP

- Nos conectamos al servidor de la BBDD original (o a cualquier otro que tenga acceso a NFS_BACKUP), nos situamos en la ruta donde esté el DMP recuperado y lanzamos el IMPORT con el comando “at now” para que si se nos desconecta la sesión, no se corte el IMPORT:

cd /NFS_BACKUP/BBDD/_trabajo_/INC-XXXXXX

- Lanzamos (ejemplo con EXPJUD de PROD02):

at now

imp USERID=SYSTEM/XXXXXX@BDADRIANOAUX BUFFER=100000 FILE=PROD02-EXPJUD.dmp LOG=import_EXPJUD_PROD02.log FROMUSER=EXPJUD TOUSER=RES_EXPJUD_PROD02 GRANTS=NO INDEXES=NO IGNORE=YES

ctrl+D (para aceptar la ejecución del comando)

- Si queremos que el IMPORT sea solamente de algunas tablas añadimos TABLES=TABLE_NAME1,TABLE_NAME2,TABLE_NAME3

3.2 Import desde IUSNAS05

Abrimos un CMD y preparamos para lanzar la restauración:

ESQUEMA ENTERO:

imp USERID=SYSTEM/XXXXXX@BDADRIANOAUX BUFFER=100000 FILE=Z:\INC_XXXXXX\PRINST.dmp LOG=Z:\INC_XXXXXX\import_PRINST_ALEJ.log FROMUSER=PRINST TOUSER=RES_PRINST_ALEJ
GRANTS=NO INDEXES=NO IGNORE=YES

* SHOW=YES (ejecuta el IMP como una simulación mostrando las sentencias que va a ejecutar)

Sólo TABLAS:

imp USERID=SYSTEM/XXXXXX@BDADRIANOAUX BUFFER=100000 FILE=Z:\INC_XXXXXX\PRINST.dmp LOG=Z:\INC_XXXXXX\import_PRINST_ALEJ.log FROMUSER=PRINST TOUSER=RES_PRINST_ALEJ TABLES=ASUNTOS,ACONTECIMIENTOS,ACO_DOC_COPIAS GRANTS=NO INDEXES=NO IGNORE=YES

- Para sacar el CREATE de algún objeto concreto que nos haya fallado en el import también podemos ejecutar en origen:

SQL>; select dbms_metadata.get_ddl('TABLE','ACONTECIMIENTOS','PRINST') FROM DUAL;
