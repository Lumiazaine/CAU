NORMATIVA WSO2

En esta página centralizaremos definiremos y estandarizaremos las reglas, buenas prácticas y criterios técnicos que deben cumplirse en el diseño, desarrollo, configuración y operación de las soluciones basadas en la plataforma WSO2. Esta normativa garantiza coherencia, seguridad, mantenibilidad y alineación con los estándares corporativos y arquitectónicos.

Se incluye, en primer lugar, el documento de normativa que se tendrá que seguir para desarrollar un servicio web en WSO2 con MicroIntegrator

  • Normativa Desarrollo Servicios Web 2.2.0 WSO2 MI

    • Se incluye el documento plantilla con la descripción de requisitos funcionales de integración de un servicio web, que será rellenado por el consumidor que desee consumir un servicio en particular.

  • Normativa Servicios web consumidor

    • Se incluye el modelo de definición de OpenAPI para los servicios que se publiquen en la plataforma WSO2.

  • Normativa OpenAPI.

    • Se incluye un enlace el cual se dirigirá a la normativa de los servicios web tanto REST como SOAP de la OTIJ, la cual todos los servicios que se publiquen en WSO2, deberán de seguir

  • Normativa servicios web REST - SOAP OTIJ.
