1.1 WINDOWS

Editar

- Las restauraciones en WINDOWS se hacen en la ruta \\iusnas05\Restauraciones, dentro de una carpeta con el número del ticket que la requiera.

Para que no haya problemas al hacer operaciones haciendo referencia a un fichero del directorio, mapearemos este recurso compartido con una letra de unidad (en mi caso elegí la Z.

1) Vamos a Mi PC > Herramientas (en el panel de navegación superior) > Conectar a unidad de red

2) Elegimos la unidad Z: y la carpeta \\iusnas05\Restauraciones y Finalizar.

- Ahora podremos operar con los ficheros recuperados a través de una ruta tipo Z:\IN0000001337612\GSE2.dmp

1.2 HP-UX

Editar

- Las restauraciones en HP-UX se hacen en la ruta /NFS_BACKUP/BBDD/_trabajo_, dentro de una carpeta con el número del ticket que la requiera.

En el proceso de restauración de ficheros de SSCC podremos elegir la ruta dentro de /NFS_BACKUP que deseemos, por lo que no requerimos ninguna operación como en el caso de WINDOWS.

2. Recuperar EMITIDOS y DMP

Editar

2.1 SEDES

Editar

Ejemplo de restauración con SE2CGSE2

- Desde la misma máquina de salto (10.241.252.119) entramos en un cmd y ejecutamos:

cmd> recover -s oragrid -c SE2CGSE2
oragrid = Es el servidor (-s) de backup para las SEDES
SE2CGSE2 = Es el cliente (-c) del que queremos coger el backup (se indicará en la petición, sino consultar con Adriano)

- Los siguientes comandos nos servirán para localizar, marcar y recuperar los ficheros que necesitemos del cliente:

recover> changetime 07/21/2018 –> Modifica la fecha en formato (mm/dd/yyyy)
recover> cd O:\GSE2\export –> Moverse por directorios
recover> ls -la –> Listar ficheros del directorio actual por nombre
recover> add GSE2.dmp –> Añadir fichero a la lista de recuperación
recover > add O:\GSE2\export –> Añade todos los ficheros dentro del directorio a la lista de recuperación
recover> list –> Mostrar ficheros de la lista para recuperar
recover> delete O:\GSE2\export\GSE2.dmp –> Eliminar ficheros de la lista para recuperar
recover> relocate Z:\INC_1322587 –> Definir directorio de recuperación
recover> destination –> Comprobar directorio de recuperación
recover> recover –> Recuperar

- Para los EMITIDOS, si no se encuentran los ficheros solicitados en la fecha, conectarse por SSH al servidor oragrid y hacer por cada directorio:
# nsrinfo justiciase4cad1.sevilla.justicia.junta-andalucia.es | grep “\\
4109142024 \\
2014009028 ”

Son importantes las \\\\ porque nsrinfo reconoce las 4 como 1. También importantes las dobles comillas.

- Para facilitar el trabajo a los compañeros de ADRIANO, dejaremos los EMITIDOS en una carpeta con el nombre de la carpeta superior a donde están dichos ficheros, es decir, para el ejemplo anterior sería crear una carpeta 2014009028 dentro de Z:\IN0000001337612
