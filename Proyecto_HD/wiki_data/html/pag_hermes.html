# 📄 Documentación Hermes

## 1️⃣ Introducción 🕒

  • Hermes es una aplicación web utilizada por la Consejería de Gobernación y Justicia para el control de presencia y permisos.
  • Los funcionarios, al entrar y salir de su puesto de trabajo, fichan a través de esta herramienta.
  • La solicitud de vacaciones y permisos también se gestiona mediante Hermes y es validada por el responsable correspondiente.

## 4️⃣ Alcance 🎯

  • - Hermes: Aplicación en producción para fichaje y solicitud de permisos por parte de todos los funcionarios de justicia.
  • - Hermes-Pre: Entorno de preproducción para pruebas de despliegues y validaciones.
  • - Hermes-Validación-Pre: Servidor de validación para usuarios, con base de datos de datos reales ofuscados.

## 5️⃣ Configuración ⚙️

  • Parada / Arranque del servicio: `systemctl start|stop tomcat-hermes.service`

Tareas programadas nocturnas: - Recálculo de saldos:

  1. Actualiza información de los usuarios.
  2. Job de base de datos programado a las 00:01.

- Sincronización con Temis:

  1. Volcado de información desde Temis (repositorio central).
  2. Ejecutada por Tomcat; configuración en `\$CATALINA_HOME/webapps/Hermes/WEB-INF/classes/schedule.properties`.

## 7️⃣ Monitorización / Gestión de eventos 📡 - Servicios monitorizados en Nagios:

  1. Acceso Hermes
  2. Ping

  4. Tomcat Nodo 02
  5. URL de la aplicación
  • Notificaciones: GrupoSistemas (todos los departamentos de sistemas)
  • Alertas: down, recovery, up
  • Procedimiento ante errores: Reiniciar Tomcat → Revisar logs → Diagnosticar incidencias.
