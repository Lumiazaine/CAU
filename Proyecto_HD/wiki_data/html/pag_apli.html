IMPORTANTE: Obligatoriedad de la adaptación de todas las aplicaciones a los certificados de curva elíptica

Obligatoriedad de la adaptación de todas las aplicaciones a los certificados de curva elíptica, resumen muy esquemático de la situación:

  • La FNMT va a dejar de emitir certificados RSA desde nov-26 y los que emita los hará con caducidad de 30/11/2026.
  • En su lugar serán ECC (más seguros y con caducidad máxima de 6 meses y tendentes a cada vez menor caducidad).
  • Los SSII que no dispongan de las siguientes versiones o superiores no funcionarán a partir del 30/11/2026:
    • @firma v6.4 (@firma v5 se apaga el 30/6/2026)
    • Autofirma Estado v1.9 (la 1.8 también funciona) Autofirma JA se debe migrar al del estado.
    • Java 1.8.0_51
      • Notific@ v5.3.1.0
      • API REST ok
      • API SOAP DEPRECADA
  • El kit de pruebas de certificados ECC ya está disponible en la web de la ADA.
  • El portafirmas corporativo está integrado con autofirma del estado 1.9
  • El nuevo certificado ECC no revoca el anterior RSA.
  • Se utilizará NAOS para consultas.

NOVEDADES ADA (Agencia Digital de Andalucía)

Se ha publicado la versión 1.3 del Sistema de Diseño de Servicios Digitales, en el siguiente enlace:

Sistema de diseño en el Portal de Desarrollo de Servicios Digitales

En esta versión se incluye una mejora importante al realizar una estandarización de la documentación de componentes, que tiene como objetivo facilitar el trabajo diario de los equipos y asegurar una experiencia más coherente en todos nuestros desarrollos.

NOVEDADES:

Documentación más clara y homogénea

A partir de ahora, cada componente se presenta manteniendo el formato de publicación web e incorpora información técnica de las aplicaciones de referencia (código HTML y llamada al Web Component). Además, cada ficha de componente incluye información útil que esperamos sirva de ayuda para su aplicación:

  • Definición y anatomía: Una descripción técnica y visual que permite entender de un vistazo cómo está construido el elemento.
  • Tipos y código: Recopilación de todas las variantes del componente, acompañadas de su código HTML y de Web Component.
  • Pautas de uso: Indicaciones prácticas para saber cómo y cuándo emplear correctamente cada componente.
  • Relaciones y versiones: Componentes relacionados y un control de versiones específico por elemento, para que siempre sepamos qué ha cambiado en cada versión.

Actualizaciones recientes de contenido

Además de avanzar en la estandarización de la documentación, hemos incorporado mejoras simplificando componentes redundantes y explicitando algunos subcomponentes que están implícitos en componentes actuales:

  • Alcance: Hemos revisado y actualizado la descripción sobre qué servicios y herramientas cubre el sistema, y también se ha renovado el listado de sitios considerados micrositios, para que refleje mejor la situación actual.
  • Estilo gráfico: El sistema visual da un paso más con varias novedades importantes:
    • Estrenamos una nueva paleta categórica.
    • Ajustamos los puntos de corte del grid para mejorar la consistencia entre dispositivos (modo responsive).
    • Actualizamos la nomenclatura tipográfica: a partir de ahora, Source Sans 3 será la tipografía de referencia en nuevos desarrollos.
  • Componentes: También hemos aplicado mejoras relevantes en la biblioteca de componentes:
    • Nuevo diseño para el Listado de enlaces.
    • Simplificación y normalización de las tipologías de Tarjetas (cards).
    • Documentación específica e individualizada para las entradas de campo.
    • Incorporación de los segmentos de ancho completo de forma individualizada: alerta, recomendador y carrusel de banners.

Además, se dejan de utilizar dos elementos:

  • Ítem de listado de recursos, que ahora se resuelve mediante Tarjetas (cards).
  • Navegación auxiliar, que pasa a integrarse directamente dentro del componente Cabecera.
