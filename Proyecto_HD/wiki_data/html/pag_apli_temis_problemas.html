ROE - Añadir Jurisdicción a organismo

Remedy 1987946


Añado la jurisdicción Civil a las 3 secciones penales de Cádiz que no la tienen. Todas las demás Audiencias de Andalucía tienen C, M y P menos estas 3.


DECLARE

NV number(20);

cursor cxo is

select x_organismo from temis.organismos_global where t_organismo like 'Audiencia Provincial - Sección _ª - Penal de Cádiz';

cursor c(xorg number) is

select distinct

apl_x_aplicacion, esq_x_esquema, t_path1, t_path2,

sysdate, l_iniciador, l_tramite, l_notificacion, l_acuse, l_firma, t_path_servacuse, t_path_rutaslexnet,

t_path_binarios, t_path_central_recibidos, t_path_central_emitidos, l_caso

from TEMIS.RED_ORGANISMO_ESQUEMA

where org_x_organismo = xorg;

BEGIN

for rxo in cxo loop

for r in c(rxo.x_organismo) loop

select TEMIS.SQ_REDORGANISMOESQUEMA.nextval into NV from dual;

insert into TEMIS.RED_ORGANISMO_ESQUEMA values (

NV, rxo.x_organismo, 'C', r.apl_x_aplicacion, r.esq_x_esquema, r.t_path1, r.t_path2,

sysdate, r.l_iniciador, r.l_tramite, r.l_notificacion, r.l_acuse, r.l_firma, r.t_path_servacuse,

r.t_path_rutaslexnet, r.t_path_binarios, r.t_path_central_recibidos, r.t_path_central_emitidos, r.l_caso);

end loop;

end loop;

END;

==== Solicitan reactivar un usuario que ha sido “anulado” por error ==== Descripción: Solicitan desde CAU u otro departamento la reactivación de un usuario que por error ha sido “anulado”, y no lo pueden reactivar con el botón “Activar” en Mantenimiento Temis: Solicitan reactivar un usuario que ha sido Nota: La “anulación” de un usuario se realiza mediante el botón “Borrar”, el cual hace un “borrado lógico” del usuario en Temis, por lo que aunque se realice dicha acción, el registro del usuario sigue existiendo en BBDD. Explicación: Los registros de los usuario de Mantenimiento Temis se encuentran en la BBDD “PROD02”, esquema “TEMIS”, tabla “USUARIOS_GLOBAL”. Los usuarios que tienen en el campo “L_ANULADO” el valor “S”, aparecerán en Mantenimiento Temis con el estado “Anulado”. Al ser anulados también pueden haber registrado el motivo de su anulación en el campo “D_MOTIVO_CESE”. Solución:1. Se accede a la BBDD “PROD02”, esquema “TEMIS”, tabla “USUARIOS_GLOBAL” y se busca el registro del usuario mediante su ID de usuario Temis (el cual suele ser el que aparece en los datos del usuario en Remedy, que es su DNI sin letra): select * from temis.usuarios_global where t_usuario=' '; 2. Si el campo “L_ANULADO” del registro del usuario tiene el valor “S”, se obtiene el valor del campo “X_USUARIO” y se hace copia de seguridad del registro: select * from temis.usuarios_global where x_usuario=' '; Importante: Si el campo “D_MOTIVO_CESE“tiene valor, también se debe modificar en el update que realizaremos en el siguiente paso. 3. Se crea el script de update que le pondrávalor nulo a los campos “L_ANULADO” y “D_MOTIVO_CESE” (este último en el caso de que tuviese algún valor): BEGIN update TEMIS.USUARIOS_GLOBAL set L_ANULADO = NULL , D_MOTIVO_CESE = NULL where x_usuario=' '; COMMIT; EXCEPTION WHEN OTHERS THEN ROLLBACK; DBMS_OUTPUT.put_line ('Error ' || SQLCODE || ' ' || SQLERRM); END; / 4. Se ejecuta el script, se comprueba que se han modificado correctamente el/los campos del registro del usuario, se adjunta el script a la incidencia y se le indica al solicitante: Se ha activado al usuario (>) en Temis.

==== Modificar identificador de un usuario ====

Nos solicitan modificar el identificador de un usuario en Temis, bien porque sea erróneo o porque el actual tiene una nomenclatura antigua (Ej: ius21863).

2. Se revisa que el identificador que se le va a establecer (“T_USUARIO”) no lo tiene ya otro usuario:


select * from TEMIS.USUARIOS_GLOBAL where T_USUARIO=' ';


Nota: El identificador que suelen tener los usuarios con nomenclatura actual es su DNI sin letra.


3. En el caso de que el identificador no lo tenga ningún otro usuario, se hace copia de seguridad del registro actual del usuario a modificar:


select * from TEMIS.USUARIOS_GLOBAL where T_USUARIO=' ';
