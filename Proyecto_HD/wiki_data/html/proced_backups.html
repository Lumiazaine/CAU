OBSERVACIONES

  • La configuración de las retenciones y Schedule recaen sobre el grupo y no sobre el cliente.
  • Todos los grupos, mencionados anteriormente están asociados a la pool SEDES_COMUNES, lo que implica que se salva todo al Data Domain 6800 IUSAPBACK04. Dicho Data Domain tiene una tarea programada de limpieza de datos expirados.
  • La clonación de reciclado de cintas se realizan bajo tareas cron por scripts desde el servidor de Networker iusslnsrsrv01.
  • De la misma forma, por script mediante tareas cron, se realizan monitorizaciones de espacio en cintas, tamaño de copias, clientes, cruce de Save sets para TPath de comunes.
  • Las cintas se guardan en los 4 armarios ignífugos que están en el CICA.
  • Los Comunes tienen políticas de reintentos de backup después del fin de semana y entre semana.
  • Este servidor además de las copias de las Sedes también tiene configurado el cliente iusnas05 para realizar copias de ISILON-NDMP.justicia.junta-andalucia.es

Monitorización-gestión de eventos

Se lleva acabo a través de correos.

Los correos de iusdesa02 llegan desde ⇒ LEGATO.NETWORKER@justicia.junta-andalucia.es

Los correos de iusslnsrsrv01 llegan desde ⇒ root IUSSLNSRSRV01 root@iusslnsrsrv01.justicia.junta-andalucia.es

Los correos de oragrid llegan desde ⇒ ORAGRID root root@oragrid.justicia.junta-andalucia.es

Todos se mandan a la cuenta de correo notificaciones.gtab.ius@juntadeandalucia.es

Los correos se dividen en dos carpetas dentro de esta cuenta de correo, una para los bootstrap y otra para todas las demás.
