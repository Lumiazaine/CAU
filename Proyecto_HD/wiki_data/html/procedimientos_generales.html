PGGTBD - Procedimientos Generales GTBD

Descripción

Esta sección incluye procedimientos comunes y reutilizables aplicables a la administración y mantenimiento de bases de datos, independientemente del entorno o sistema específico. Se documentan buenas prácticas, pasos detallados y operaciones frecuentes que deben seguirse de forma estandarizada.

Procedimientos incluidos

  • Cambio de contraseñas en BBDD
    1. Pasos para modificar credenciales de usuarios SYS, SYSTEM y aplicaciones.
    2. Validación posterior y propagación en entornos replicados.
  • Conexión a instancias Oracle y SQL Server
    1. Métodos de acceso (sqlplus, Toad, SQL Developer, etc.).
    2. Acceso mediante VPN y revisión de listener.
  • Reinicio controlado de instancia
    1. Procedimiento para parar y levantar una instancia Oracle.
    2. Consideraciones de red, dependencias y notificación a usuarios.
  • Creación y asignación de tablespaces
    1. Alta de nuevos espacios de almacenamiento.
    2. Asignación a usuarios y control de autoextend.
  • Creación de usuarios
    1. Script estándar para nuevos usuarios en Oracle/SQL Server.
    2. Asignación de roles, privilegios y perfiles.
  • Control de bloqueos
    1. Identificación de sesiones bloqueantes.
    2. Procedimiento para desbloqueo y trazabilidad.
  • Gestión de trazas
    1. Localización de ficheros trace y alert logs.
    2. Análisis y depuración de errores.
  • Validación post-mantenimiento
    1. Checklist de puntos a revisar tras tareas de impacto.
    2. Verificación de servicios, conexiones y rendimiento.
  • Revisión de logs de auditoría
    1. Análisis de accesos y cambios relevantes.
    2. Control de accesos indebidos o no planificados.
